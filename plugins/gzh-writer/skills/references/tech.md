# 技术类文章写作指南

## 写作风格

1. **专业性**：确保内容准确无误，不要脱离用户提供的参考资料。
2. **清晰易懂**：使用简洁明了的语言，避免过于复杂的术语，必要时进行解释。
3. **简洁明了**：每个段落避免过长，段落之间加空行分隔，提升可读性。
4. **结构清晰**：使用标题和小标题划分内容，便于读者快速浏览和理解文章结构，但尽量不要使用三级及以上层级。
5. **不用注明参考资料**：文章中不需要注明参考资料的出处，除非用户有特殊要求。

## 写作步骤

## 1. 阅读材料

读取用户存放在 `refs` 文件夹下的技术类参考资料，理解内容和要点。

## 2. 确定主题

一般用户会提供写作主题，如果不明确，请根据参考资料提供几个可能的主题供用户选择。

## 3. 确定受众

根据文章主题和参考资料，确定目标读者群体，如初学者、中级开发者或高级架构师等。

## 4. 制定大纲

根据主题与参考材料，制定文章大纲，。根据实际情况采用如下策略生成大纲：

1. 解决问题导向：采用“问题-解决方案-总结”的结构，适合技术教程类文章。
2. 过程导向：采用“背景-步骤-结果”的结构，适合技术实现类文章。
3. 案例分析导向：采用“案例介绍-分析-结论”的结构，适合技术案例类文章。
4. 知识分享导向：采用“概念介绍-应用场景-实践建议”的结构，适合技术分享类文章。

不要死板地按照上述标题生成大纲，可以根据具体内容灵活调整标题和结构，但要确保逻辑清晰，层次分明。

## 5. 撰写内容

根据大纲逐步撰写文章内容，确保每个部分内容充实且紧扣主题。注意以下几点：

- 引入部分：一般作者会提供引言内容，如果没有，请根据主题撰写简短的引言，介绍文章背景和目的。
- 主体部分：根据大纲逐步展开内容，确保逻辑清晰，段落分明。
- 最后： 根据引入部分内容及主题内容，撰写总结段落，最好前后呼应

## 6. 审校与修改

完成初稿后，进行审校和修改，确保内容准确无误，语言流畅自然。重点检查以下方面：

- 事实准确性：确保技术细节和数据正确无误。
- 逻辑连贯性：确保段落和章节之间逻辑清晰，过渡自然。
- 语言表达：确保用词准确，句子通顺，避免语法错误。
